# 职引官开发计划

## 当前阶段

职引官已经进入“AI 面试工作台 + 真实数据闭环”阶段。项目不再只做静态演示，当前重点是让简历、岗位、面试问答、复盘报告和成长记录进入稳定的 MySQL 数据链路。

## 已完成

- 基础工程：Express + MySQL + 单页前端，静态资源已拆分到 `public/`。
- AI 面试：支持岗位/JD 生成题目、面试官风格、用户答题、AI 追问、复盘报告、能力雷达图和历史记录。
- 桌面工作台：AI 面试作为默认入口，形成“简历 + 自定义面试 + 视频入口 + 面试官风格 + 最近面试”的工作台布局。
- 简历模块：支持纯文本/txt 简历上传、保存原文、AI 解析、本地兜底解析、摘要展示，并在题目生成/追问/复盘时携带 `resumeId` 和 `resumeSummary`。
- 本地数据库：已补充 `resumes` 表，并提供 `npm run db:start`、`npm run test:db`。

## 主要缺口

- P0：面试流程仍以一次性 `interview_records` 为主，缺少 session 和逐轮问答表。
- P1：岗位模板库尚未独立，题目生成仍主要依赖 JD、知识库和硬编码 fallback。
- P1：简历解析是第一版，暂不支持 pdf/docx 后端解析和多简历管理。
- P2：知识库检索仍是基础关键词匹配。
- P2：成长曲线尚未基于真实历史报告自动计算。
- P3：视频面试仍是演示入口，尚未复用真实面试流程。

## 下一阶段优先级

1. 面试 session 化：新增 `interview_sessions`、`interview_turns`，支持创建会话、逐题保存、追问入库、结束报告和刷新恢复。
2. 岗位模板库：新增 `job_templates`，让题目生成结合“岗位模板 + JD + 简历 + 知识库”。
3. 真实成长曲线：新增用户成长统计接口，个人中心展示真实总分趋势、能力维度、高频短板和训练建议。
4. 简历增强：支持 multipart txt、pdf/docx 文本提取、多简历管理和岗位匹配度评分。
5. 视频流程打通：视频面试复用 session、题目、语音输入/朗读和复盘报告。

## 技术原则

- 保持 Express + MySQL + 原生单页前端结构，不做大型框架重构。
- 数据库脚本可重复执行，优先使用 `CREATE TABLE IF NOT EXISTS`。
- 所有 AI 能力都必须有本地 fallback。
- 前端继续保持当前 AI 面试工作台风格。
- 每完成一个完整模块，更新 `CBY-history.md`。
- 修改后至少运行：
  - `node --check src/server.js`
  - `node --check src/prompts.js`
  - `node --check public/js/app.js`
  - `npm run test:db`

## 下一步开发 Prompt

```text
请基于当前“职引官”项目，优先实现“面试流程 session 化”功能。

目标：
1. 将当前一次性 interview_records 保存方式升级为“面试会话 + 逐轮问答 + 结束报告”的数据闭环。
2. 新增 interview_sessions 和 interview_turns 表。
3. 后端支持创建会话、读取会话、保存每轮问题/回答、生成追问、结束面试并保存报告。
4. 前端 AI 面试流程改为使用 sessionId 推进，每答一题就保存当前轮次。
5. 页面刷新或中断后，能通过 sessionId 恢复已保存的问题、回答和当前进度。
6. 最近面试列表优先展示 session 化后的记录，同时兼容旧 interview_records。

建议接口：
- POST /api/interview/sessions
- GET /api/interview/sessions/:id
- GET /api/interview/sessions/user/:userId
- POST /api/interview/sessions/:id/turns
- POST /api/interview/sessions/:id/followup
- POST /api/interview/sessions/:id/finish

要求：
- 继续使用 Express + MySQL + 单页前端结构。
- session 保存 userId、position、company、jd、interviewerId、resumeId、resumeSummary、questions、status。
- turn 保存 sessionId、questionIndex、question、answer、kind、aiSource。
- finish 复用现有报告生成逻辑，保存 totalScore、aiReport、finishedAt，并将 session 状态改为 finished。
- AI 失败时使用本地 fallback，不能让页面不可用。
- 保持当前 AI 面试工作台风格。
- 完成后运行语法检查和数据库连接验证，并更新 CBY-history.md。
```
