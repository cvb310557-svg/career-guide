# 职引官开发计划

## 当前阶段

职引官已经进入“前端正式面试闭环优先”阶段。当前第一目标不是追求面试效果完全成熟，而是确保用户可以从前端稳定启动一次正式面试，完成答题、保存 session、生成复盘报告，并能在最近面试/历史记录中回看。2026-05-13 已完成第一轮最小闭环增强，下一步应以真实浏览器联调和异常场景验收为主。

## 已完成

- 基础工程：Express + MySQL + 单页前端，静态资源已拆分到 `public/`。
- AI 面试：支持岗位/JD 生成题目、面试官风格、用户答题、AI 追问、复盘报告、能力雷达图和历史记录。
- 面试 session 化：新增 `interview_sessions`、`interview_turns`，支持创建会话、逐题保存、追问入库、结束报告和刷新恢复。
- 桌面工作台：AI 面试作为默认入口，形成“简历 + 自定义面试 + 视频入口 + 面试官风格 + 最近面试”的工作台布局。
- 简历模块：支持纯文本/txt 简历上传、保存原文、AI 解析、本地兜底解析、摘要展示，并在题目生成/追问/复盘时携带 `resumeId` 和 `resumeSummary`。
- 岗位模板库：新增 `job_templates` 表和内置岗位模板，支持列表、详情、关键词检索；热门岗位优先来自模板库，题目生成结合“岗位模板 + JD + 简历 + 知识库”，session 保存 `templateId/templateSummary`。
- 代码拆分：后端已拆出 `utils`、`resumeService`、`jobTemplateService`、`interviewService`，前端已拆出 `app-core.js`，便于继续迭代正式面试流程。
- 本地数据库：已补充 `resumes` 表，并提供 `npm run db:start`、`npm run test:db`。
- 正式面试最小闭环：模板快速开始和自定义岗位开始都能进入题目生成、session 创建、逐题保存、结束复盘、历史回看的流程；AI 题目生成失败时使用本地模板题库继续。
- 历史状态提示：最近面试/全部历史会把未完成 session 显示为“继续答题”，完成记录才打开复盘报告。

## 主要缺口

- P0：需要用真实浏览器和真实本地 MySQL 完整验收从选择岗位/模板、生成题目、创建 session、逐题答题、完成复盘、历史回看这一整条链路。
- P0：需要补充更系统的异常场景联调，包括模型超时、数据库短暂不可用、刷新恢复、未登录练习和无历史记录。
- P0：结束面试失败时还需要更强的重试/恢复入口，避免报告生成或数据库写入异常时用户不清楚下一步。
- P1：岗位模板库已完成第一版，后续可继续补充模板运营后台、更多行业模板和模板效果统计。
- P1：简历解析是第一版，暂不支持 pdf/docx 后端解析和多简历管理。
- P2：知识库检索仍是基础关键词匹配。
- P2：成长曲线尚未基于真实历史报告自动计算。
- P3：视频面试仍是演示入口，尚未复用真实面试流程。

## 下一阶段优先级

1. 正式面试闭环浏览器验收：用真实账号从模板快速开始和自定义岗位开始各跑完一次，并确认 `interview_sessions/interview_turns` 写入、状态 `finished` 和历史报告打开。
2. 异常场景补强：完善模型超时、数据库短暂异常、未登录练习、无历史记录和结束失败后的重试提示。
3. session 体验增强：完善结束前确认、未完成会话列表管理、异常恢复提示和刷新后继续答题。
4. 报告可用性增强：先保证报告稳定生成，再逐步提升报告质量、岗位模板利用率和简历引用质量。
5. 真实成长曲线：在正式面试闭环稳定后，再新增用户成长统计接口和个人中心真实趋势展示。
6. 简历增强与视频流程：后续支持 pdf/docx、多简历管理、视频面试复用真实 session 流程。

## 第一目标验收标准

- 用户可以在前端 AI 面试页选择热门岗位模板或进入自定义面试。
- 点击开始后，无论 AI 是否可用，都能得到一组面试题。
- 登录用户开始面试时会创建 `interview_sessions` 记录。
- 每道题的回答会保存到 `interview_turns`，切题或刷新后不丢失关键进度。
- 用户可以完成最后一题并生成复盘报告；AI 不可用时使用本地报告 fallback。
- 完成后的记录能出现在“最近面试”或历史列表中，并可打开查看报告。
- 前端不出现白屏；失败状态要给出可继续操作的提示。

## 技术原则

- 保持 Express + MySQL + 原生单页前端结构，不做大型框架重构。
- 数据库脚本可重复执行，优先使用 `CREATE TABLE IF NOT EXISTS`。
- 所有 AI 能力都必须有本地 fallback。
- 前端继续保持当前 AI 面试工作台风格。
- 每完成一个完整模块，更新 `CBY-history.md`。
- 修改后至少运行：
  - `node --check src/server.js`
  - `node --check src/prompts.js`
  - `node --check public/js/app.js`
  - `npm run test:db`
